# 기술 스택

## 프론트엔드

### React 19
- **버전**: 19.2.1
- **용도**: UI 컴포넌트 및 상태 관리
- **특징**: 
  - 함수형 컴포넌트 및 Hooks 사용
  - 실시간 데이터 업데이트 처리
  - 컴포넌트 기반 아키텍처

### TypeScript
- **버전**: 5.8.2
- **용도**: 타입 안정성 및 개발 생산성 향상
- **특징**:
  - 정적 타입 검사
  - 인터페이스 및 타입 정의
  - IDE 자동완성 지원

### Vite
- **버전**: 6.2.0
- **용도**: 빌드 도구 및 개발 서버
- **특징**:
  - 빠른 개발 서버 (HMR)
  - 최적화된 프로덕션 빌드
  - ES 모듈 기반 번들링

### Tailwind CSS
- **용도**: 유틸리티 기반 CSS 프레임워크
- **특징**:
  - 반응형 디자인
  - 커스텀 색상 및 스타일
  - 빠른 스타일링

## 백엔드 및 데이터베이스

### Firebase Realtime Database
- **용도**: 실시간 데이터 저장 및 동기화
- **특징**:
  - 실시간 데이터베이스
  - 자동 동기화
  - RESTful API 및 WebSocket 지원
- **데이터 구조**: JSON 기반 NoSQL 데이터베이스

### Firebase SDK
- **버전**: 12.6.0
- **용도**: Firebase 서비스 통합
- **주요 기능**:
  - Realtime Database 읽기/쓰기
  - 실시간 리스너 구독
  - 데이터 마이그레이션

## 프로젝트 구조

```
clinic-waiting-system/
├── components/          # React 컴포넌트
│   ├── BannerSlider.tsx
│   ├── Button.tsx
│   ├── ConfirmModal.tsx
│   ├── Header.tsx
│   ├── NoticeBanner.tsx
│   ├── PatientCard.tsx
│   └── Toast.tsx
├── lib/                # 라이브러리 및 유틸리티
│   ├── firebase.ts
│   ├── firebase.config.ts
│   └── firebase.config.example.ts
├── docs/               # 문서
│   ├── 기능_명세.md
│   ├── 기술_스택.md
│   ├── 파이어베이스_스키마.md
│   └── 배포_가이드.md
├── App.tsx             # 메인 앱 컴포넌트
├── index.tsx           # 앱 진입점
├── index.html          # HTML 템플릿
├── types.ts            # TypeScript 타입 정의
├── constants.ts        # 상수 정의
├── package.json        # 프로젝트 설정 및 의존성
├── tsconfig.json       # TypeScript 설정
├── vite.config.ts      # Vite 설정
└── vercel.json         # Vercel 배포 설정
```

## 빌드 및 실행

### 개발 환경
```bash
npm install          # 의존성 설치
npm run dev          # 개발 서버 실행 (http://localhost:5173)
```

### 프로덕션 빌드
```bash
npm run build        # 빌드 실행 (dist/ 디렉토리에 생성)
npm run preview      # 빌드 결과 미리보기
```

## 성능 최적화

1. **코드 스플리팅**: Vite의 자동 코드 스플리팅 활용
2. **이미지 최적화**: 필요 시 이미지 최적화 도구 사용
3. **번들 크기 최적화**: Tree-shaking 및 미사용 코드 제거
4. **캐싱**: Firebase 데이터 캐싱으로 불필요한 요청 감소
5. **반응형 디자인**: 모바일 우선 설계로 성능 향상

## 보안

1. **환경 변수**: 민감한 정보는 환경 변수로 관리
2. **Firebase 보안 규칙**: 데이터베이스 접근 제어
3. **타입 안정성**: TypeScript로 런타임 오류 방지
4. **입력 검증**: 사용자 입력 데이터 검증

## 확장성

- **컴포넌트 기반 아키텍처**: 재사용 가능한 컴포넌트 구조
- **타입 시스템**: TypeScript로 유지보수성 향상
- **모듈화**: 기능별로 분리된 모듈 구조
- **설정 관리**: 중앙화된 설정 관리
